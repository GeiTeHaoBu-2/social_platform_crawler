# 类重构检查清单

> 在重构/拆分大型类时，必须按此清单逐项检查

## ✅ 第一阶段：分析阶段（重构前必须完成）

### 1.0 变量名映射检查
```markdown
□ 如果要修改变量名，记录新旧映射关系
□ 使用IDE的全局替换功能
□ 替换后搜索旧变量名确认无遗漏
□ 检查日志字符串中的变量名
```

### 1.1 收集接口信息
```markdown
□ 列出原类所有公共方法（包括属性）
□ 记录每个方法的完整签名（参数、返回值、类型注解）
□ 找出所有调用该类的代码文件
□ 记录每个调用点使用的具体方法
```

### 1.2 确定兼容性要求
```markdown
□ 哪些方法必须保持完全兼容？
□ 哪些方法可以修改签名？
□ 哪些调用点需要同步更新？
□ 是否有向后兼容的要求？
```

## ✅ 第二阶段：重构阶段（编码时严格执行）

### 2.1 方法迁移检查
```markdown
□ 每个公共方法都迁移了吗？
□ 方法签名是否完全一致？
  - 参数名称
  - 参数类型
  - 默认值
  - 返回值类型
□ 异常处理逻辑是否保留？
```

### 2.2 属性迁移检查
```markdown
□ 所有公共属性都迁移了吗？
□ 属性访问方式是否一致？
□ 是否需要添加@property装饰器？
```

## ✅ 第三阶段：验证阶段（重构后必须执行）

### 3.1 代码层面验证
```markdown
□ 运行 grep/search 查找所有调用点
□ 对比新旧类的方法列表
□ 检查每个调用点的参数匹配
□ 检查导入语句是否需要更新
```

### 3.2 测试层面验证
```markdown
□ 运行单元测试
□ 运行集成测试
□ 如果可能，运行完整流程测试
□ 检查是否有 AttributeError
```

## ✅ 第四阶段：文档阶段

### 4.1 更新文档
```markdown
□ 更新类的文档字符串
□ 记录破坏性变更
□ 记录向后兼容的接口
□ 添加迁移指南（如需要）
```

## 🔴 常见错误（必须避免）

| 错误类型 | 示例 | 预防措施 |
|---------|------|---------|
| 方法遗漏 | `get_items_for_db_write()` 缺失 | 对比方法列表 |
| 参数不匹配 | `process_items()` 缺少参数 | 检查调用点 |
| 返回值不匹配 | 返回tuple vs list | 检查类型注解 |
| 属性访问错误 | `_attr` 改为 `attr` | 检查访问权限 |
| 导入路径错误 | 模块移动后导入失败 | 检查所有导入 |

## 🔧 实用工具命令

```bash
# 1. 查找所有调用点
grep -r "ClassName(" --include="*.py" .

# 2. 列出类的所有方法
grep -E "^\s+def " common/process/old_class.py

# 3. 对比方法差异
diff <(grep -E "^\s+def " old_class.py | sort) \
     <(grep -E "^\s+def " new_class.py | sort)
```
