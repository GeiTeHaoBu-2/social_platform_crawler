### 3.2 Kafka 发送确认回调

**问题描述**: Kafka 异步发送没有回调，消息丢失无法感知。

**修改内容**:
```python
def _on_send_success(self, record_metadata):
    self._send_success_count += 1

def _on_send_error(self, excp):
    self._send_fail_count += 1
    logger.error(f"Kafka消息发送失败: {excp}")

def send(self, topic: str, message: dict, key: Optional[str] = None):
    future = self.producer.send(topic, key=key, value=message)
    future.add_callback(self._on_send_success)
    future.add_errback(self._on_send_error)
```

**修改文件**:
- `common/transmit/kafka_producer.py`

## 5. 新功能：热搜加速度计算 (2026-04-11)

### 5.1 功能说明

新增热搜加速度计算功能，用于分析热搜热度/排名的变化趋势。

**加速度公式**：
- `heatVelocity = (当前热度 - 上次热度) / 时间差(分钟)`
- `rankVelocity = (当前排名 - 上次排名) / 时间差(分钟)`

### 5.2 数据库表结构修改

**修改 `weibo_trend` 表字段类型**：
```sql
ALTER TABLE `weibo_trend` 
  MODIFY COLUMN `heat_velocity` FLOAT NULL DEFAULT 0 COMMENT '热度加速度(每分钟热度变化)',
  MODIFY COLUMN `rank_velocity` FLOAT NULL DEFAULT 0 COMMENT '排名加速度(每分钟排名变化，负数表示排名上升)';
```

### 5.3 新建模块

**`common/process/velocity_calculator.py`**：
- `VelocityCalculator` 类：计算热搜加速度
- 从 Redis 读取上一轮数据，计算当前轮与上一轮的差值
- 支持多平台扩展

### 5.4 Kafka 消息格式

**修改后的消息格式（驼峰命名）**：
```json
{
  "itemId": "e99a18c428cb38d5f260853678922e03",
  "rankPos": 1,
  "title": "某热搜标题",
  "url": "https://s.weibo.com/...",
  "heat": 5000000,
  "sentimentScore": 0.75,
  "typeName": "娱乐",
  "topicName": "明星八卦",
  "heatVelocity": 100000.0,
  "rankVelocity": -1.0,
  "crawlTime": 1712841600000
}
```

| 文件 | 操作 | 说明 |

| `docs/social_platforms_analysis.sql` | 修改 | weibo_trend 表字段类型改为 FLOAT |

### 5.6 数据流

```
爬取数据
    ↓
清洗
    ↓
加速度计算（从Redis读上一轮数据）
    ↓
Redis缓存（保存当前数据供下一轮使用）
    ↓
LLM分析
    ↓
MySQL写入:
  - base表（新增热搜）
  - analysis表（分析结果）
  - trend表（全量趋势数据）← 新增
    ↓
Kafka发送（包含velocity字段）
```

### 5.7 边界情况处理

| 场景 | 处理方式 |
|------|----------|
| 首次运行（无上一轮数据） | 所有 velocity = 0 |
| 热搜首次上榜 | velocity = 0 |
| 时间差 ≤ 0 | velocity = 0（避免除零） |
| Redis 连接失败 | velocity = 0，继续执行 |
